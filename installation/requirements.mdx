### QAT Software Acceleration

QAT_SW acceleration is supported on platforms starting with:

- [3rd Generation Intel Xeon Scalable Processors](https://www.intel.com/content/www/us/en/products/docs/processors/xeon/3rd-gen-xeon-scalable-processors-brief.html) and later

<Note>
QAT_SW provides software-based cryptographic acceleration using optimized libraries on compatible Intel processors.
</Note>

## Software Requirements

Successful operation requires QAT Hardware drivers (for QAT_HW) or optimized software libraries (for QAT_SW), along with a compatible crypto library.

### QAT Software Libraries

Required for QAT_SW acceleration:

| Library | Version | Purpose |
|---------|---------|----------|
| [Intel Crypto Multi-buffer](https://github.com/intel/cryptography-primitives/tree/develop/sources/ippcp/crypto_mb) | IPP Crypto v1.2.0, 2021.10 | Vectorized crypto operations |
| [Intel Multi-Buffer crypto for IPsec](https://github.com/intel/intel-ipsec-mb) | v2.0 | IPsec crypto acceleration |

<Note>
For BoringSSL support, use IPP Crypto 2021.10 specifically.
</Note>

### Build Prerequisites

Required autotools for building from source:

- autoconf
- automake
- libtool
- pkg-config

<Warning>
Ensure all prerequisites are installed before running `autogen.sh` to configure the build.
</Warning>

## Version Compatibility

This release has been validated on the versions listed above and is expected to work on:
- All major Linux distributions
- FreeBSD 13.2 and 14
- Latest versions from the linked repositories

<Note>
For the most up-to-date compatibility information, refer to the [Intel QuickAssist Technology](https://www.intel.com/content/www/us/en/developer/topic-technology/open/quick-assist-technology/overview.html) documentation.
</Note>
